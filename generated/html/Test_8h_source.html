50 // Class representing a single step in a test
51 class TestStep {
52 public:
53 // The different types of teststeps.
54 enum class Type {
55 // Step that assigns a value to a condition. Does not cause the game to step.
56 APPLY,
57 // Step that verifies if a certain condition is true. Does not cause the game to step.
58 ASSERT,
59 // Branch with a label to jump to when the condition in child is true.
60 // When a second label is given, then the second is to jump to on false.
61 // Does not cause the game to step, except when no step was done since last BRANCH or GOTO.
62 BRANCH,
63 // Step that calls another test to handle some generic common actions.
64 CALL,
65 // Step that adds game-data, either in the config-directories or in the game directly.
66 INJECT,
67 // Step that performs input (key, mouse, command). Does cause the game to step (to process the inputs).
68 INPUT,
69 // Label to jump to (similar as is done in conversations). Does not cause the game to step.
70 LABEL,
71 // Instructs the game to set navigation / travel plan to a target system
72 NAVIGATE,
73 };
