26// Class mapping key presses to specific commands / actions. The player can
27// change the mappings for most of these keys in the preferences panel.
28// A single Command object can represent multiple individual commands, e.g.
29// everything the AI wants a ship to do, or all keys the player is holding down.

71 // This command is given in combination with JUMP or LAND and tells a ship
72 // not to jump or land yet even if it is in position to do so. It can be
73 // given from the AI when a ship is waiting for its parent. It can also be
74 // given from the player/input engine when the player is preparing his/her
75 // fleet for jumping or to indicate that the player is switching landing
76 // targets. (There is no explicit key mapped to this command.)
77 static const Command WAIT;
78 // This command from the AI tells a ship that if possible, it should apply
79 // less than its full thrust in order to come to a complete stop.
80 static const Command STOP;
81 // Modifier command, usually triggered by shift-key. Changes behavior of
82 // other commands like NEAREST, TARGET, HAIL and BOARD.
83 static const Command SHIFT;
